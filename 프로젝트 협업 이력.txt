프로젝트 협업 이력

영민


이번 Planit 프로젝트에서 저는 팀장으로서 전체 개발 방향을 총괄하고, 팀원 간의 일정과 역할을 조율했습니다.
초반에는 각자 맡은 영역을 충실히 수행하며 순조롭게 진행되었지만,
중간에 국가정보자원관리원 화재 사고로 인해 사용 예정이던 주요 공공 API가 중단되는 예기치 못한 상황이 발생했습니다.

당시 프로젝트의 핵심 기능이 멈추는 위기 속에서도,
저를 포함한 유능한 팀원들이 끝까지 포기하지 않고 함께 대안을 모색했습니다.
빠르게 회의를 열어 문제 상황을 공유하고, API 대체 방안을 검토하며 개발 방향을 재정비한 결과,
짧은 시간 안에 시스템을 복구하고 프로젝트를 성공적으로 마무리할 수 있었습니다.

이번 경험을 통해 팀원들과의 협력이 곧 최고의 문제 해결력이라는 것,
리더로서 침착하게 팀을 이끌어가는 자세의 중요성을 깊이 배웠습니다.
이 과정을 통해 기술뿐 아니라 조직 내 소통과 리더십의 역량까지 함께 성장할 수 있었습니다

종윤


AI 기반 여행 플래너 시스템 프로젝트에서 저는 Full-Stack 기술 리더로서 핵심적인 역할을 수행했습니다. 기술 스택은 Spring Boot 기반의 백엔드와 React 기반의 프론트엔드를 중심으로, OpenAI, TourAPI, 지도 API 연동 및 전체 데이터 흐름 아키텍처 구축을 주도했습니다. 프로젝트 전반에 걸쳐 기술적 기준점을 제시하고, 팀의 의사결정을 이끄는 역할을 수행하며 시스템의 초석을 다졌습니다.

최대 위기 상황: TourAPI 중단 및 전략적 대응
프로젝트 막바지에 국가정보자원관리원 화재로 인한 TourAPI 호출 전면 차단이라는 예상치 못한 위기에 직면했습니다. 이로 인해 여행지 검색 및 추천 등 핵심 기능이 모두 마비되는 심각한 상황이었으나, 저는 이를 단순 복구 대신 시스템 구조 재정립의 기회로 판단했습니다.

저는 즉각적으로 의존성 제거 전략을 수립하고 실행에 옮겼습니다.

독립적 데이터베이스 구축: 외부 API에 기반한 구조를 과감히 포기하고, 시스템의 독립성을 확보하기 위한 자체 여행지 데이터베이스(DB) 구축을 최우선 목표로 설정했습니다.

데이터 파이프라인 설계: 공공 데이터를 기반으로 방대한 여행지 데이터를 수집하는 파이프라인을 주도적으로 설계했습니다. 이후 수집된 데이터에 대한 중복 제거, 카테고리 구조화, 검색 최적화 인덱싱 작업을 통해 데이터의 품질과 활용도를 극대화했습니다.

구조 전환을 통한 성능 고도화: 기존 API 요청 방식에서 DB 직접 조회 방식으로 핵심 로직을 전환했습니다. 이로 인해 응답 속도가 크게 향상되었을 뿐 아니라, 외부 환경에 흔들리지 않는 시스템 안정성을 확보할 수 있었습니다.

협업 체계 재정비: 긴급 상황 속에서도 개발 규칙을 표준화하고 리스크 공유 및 커뮤니케이션 흐름을 재구성하며, 팀 전체가 방향성을 잃지 않도록 강력하게 리드했습니다.

결과적으로 저희 팀은 외부 변수에 휘둘리지 않고, 오히려 독립성과 확장성을 갖춘 데이터 중심 구조로 시스템을 성공적으로 고도화했습니다. 이 경험은 제게 단순히 코드를 작성하는 개발자를 넘어, 예상치 못한 위기 속에서도 프로젝트를 끝까지 밀어붙이는 책임감 있는 기술 리더이자, 시스템의 근본적인 문제 해결 능력을 갖춘 팀 플레이어로 성장하는 중요한 계기가 되었습니다.

재원


이번 AI 기반 여행 플래너 시스템 프로젝트는 저에게 기술적 성숙도와 책임감을 동시에 시험했던, 가장 기억에 남는 경험입니다.

저는 시스템 전체의 기술적 책임을 맡아 Full-Stack 개발을 완수했습니다. Spring Boot 백엔드부터 React 기반 프론트엔드 통합까지, OpenAI와 지도 API 연동을 직접 주도하며 기능적 완성도를 높였습니다.

하지만 개발한지 약 일주일이 지난 후 기능을 어느 정도 완성했을 때, 국가정보자원관리원 화재로 인해 핵심 API 사용이 불가능해지는 치명적인 위기에 직면했습니다. 팀 전체가 좌절할 수 있는 상황이었지만, 저는 긍정적인 태도와 주도적인 소통으로 팀의 분위기를 붙잡았습니다. 저는 소통 허브를 자처하며 팀원들의 아이디어를 취합하고, 함께 대안 솔루션을 찾아 실행에 옮겼습니다.

결과적으로, 저희 팀은 위기를 피하지 않고 정면 돌파했습니다. 단순히 복구하는 것을 넘어, 기존 시스템보다 더 안정적이고 발전된 구조로 시스템을 재구축하는 데 성공했습니다. 저는 이 경험을 통해 기술적 난관뿐만 아니라, 팀의 위기 상황에서도 흔들림 없이 목표를 완수하는 책임감이 무엇인지를 깊이 배웠습니다. 또한, 개발 규칙 표준화와 체계적인 리스크 관리를 통해 팀 효율을 높이며, 함께 성장하는 협업의 가치를 실현할 수 있었습니다.

정현


이번 Planit 프로젝트에서 저는 축제/맛집 기능의 설계·구현과 프론트-백엔드 연결 품질 관리를 맡았습니다.
초반엔 데이터 정규화와 이미지 경로 체계를 다듬고, 카카오 로컬 API(키워드 장소 검색) 기반의 주변 맛집 추천을 안정화하면서 순조롭게 진행되었습니다. 중간에 예기치 못한 장애(공공 API 중단, 이미지 파일명 불일치, CORS·인증 필터 충돌 등)가 연달아 발생했지만, 문제를 회피하지 않고 원인부터 찾아 해결했습니다.  그 결과

이미지 파일명·경로 정규화 스크립트 & SQL 패치로 메인 썸네일 불일치 해소

JWT 필터 예외 처리와 프록시 설정 정리로 403/CORS 이슈 제거

같은 조치를 빠르게 적용해 핵심 흐름을 복구했습니다.

이번 경험을 통해 느낀 점은 두 가지입니다. 첫째, 협업의 힘—서로의 작업 맥락을 투명하게 공유하고 바로바로 대안(대체 API, 임시 완충 로직)을 실험했기에 짧은 시간에 서비스를 되살릴 수 있었습니다. 둘째, 리스크를 구조적으로 줄이는 습관—데이터/파일 체계, 에러 핸들링, 네트워크 레이어를 표준화하면 비슷한 문제가 재발해도 영향 범위를 최소화할 수 있음을 배웠습니다.

프로젝트를 마치며, 기능을 “만드는 것”을 넘어 유지·복구 가능한 품질로 끌어올리는 역량과 팀 안팎의 소통/조율 능력이 함께 성장했다고 느낍니다. 다음 프로젝트에서는 이번에 정리한 체크리스트와 템플릿을 초기부터 적용해 더 빠르고 안정적으로 결과를 내겠습니다.
마지막으로 유능한 팀원과 팀장 덕분에 어려운 상황에서도 즐겁게 개발했습니다 감사합니다.

태영


이번 최종프젝(Planit 고도화) 에서 저는 지도 기반 카테고리 UI·UX 개선과 모바일 반응형 설계를 중심으로 프론트엔드 개발을 담당했습니다.
초반엔 Kakao Map API를 활용해 관광지·음식점·카페·숙소·주차장 등 카테고리별 마커 표시와 토글 기능을 구현하며 전체적인 지도 구조를 잡았습니다. 이후에는 PC·모바일 환경의 차이로 인한 드래그/터치, 색상 반영, 테두리 불투명도 문제를 해결하며 완성도를 높였습니다.

중간에 예상치 못한 이슈들도 많았습니다.
모바일에서 선택 해제 시 색상 반영이 지연되거나, 지도 터치가 막히는 문제, PC/모바일 전환 시 상태 불일치 등 세밀한 부분에서 충돌이 발생했지만, DOM 이벤트 흐름과 CSS 상태를 하나씩 추적하며 원인을 찾아 해결했습니다.
그 결과,

카테고리 토글 시 실시간 색상·테두리 반영 로직 개선

모바일 지도 드래그/터치 병행 처리 안정화

Grid 기반 반응형 구조로 레이아웃 일관성 확보
를 달성했습니다.

이번 경험을 통해 크게 두 가지를 배웠습니다.
첫째 사용자 경험의 세부 차이가 완성도를 만든다는 것—작은 인터랙션 하나가 서비스 신뢰도를 결정짓는다는 점을 느꼈습니다.
둘째 협업의 속도는 공유의 투명성에서 나온다는 것—CSS 구조나 이벤트 수정사항을 팀원과 즉시 공유하고, 공용 컴포넌트 스타일을 통일하면서 충돌을 최소화할 수 있었습니다.

프로젝트를 마치며, 단순히 화면을 “보이게 만드는 것”을 넘어 플랫폼 전반의 UX 품질을 관리하는 시야를 얻게 되었습니다.
다음 프로젝트에서는 이번에 정리한 UI 가이드라인과 공용 CSS 구조를 초기 단계부터 적용해, 더 빠르고 일관된 인터페이스 개발을 이끌고 싶습니다.
무엇보다 어려운 순간마다 함께 디버깅하고 방향을 조율해준 팀원들 덕분에, 끝까지 집중하며 즐겁게 마무리할 수 있었습니다.
